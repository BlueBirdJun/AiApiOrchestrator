🤖 Agent 모드 동작 구조

AI API Orchestrator의 Agent 모드가 어떻게 작동하는지 알아보세요!

🎯 핵심 포인트: Agent 모드는 AI 모델을 사용하지 않고 순수 C# 코드로 구조화된 응답을 생성합니다!

📋 Agent 모드 처리 과정

1
사용자 입력 받기
사용자가 "서울 날씨 알려줘"라고 입력
⬇️
2
AgentAiService 호출
Agent 모드가 선택되어 AgentAiService.ProcessPromptAsync() 실행
⬇️
3
키워드 분석 (ApiOrchestrator)
"날씨" 키워드를 감지하여 GetWeather API 선택
키워드 매핑: "날씨" → GetWeather API
⬇️
4
Sample API 호출 (SampleApiClient)
내부 HTTP 호출: GET http://localhost:5152/api/sample/weather?location=서울
HTTP 요청 → SampleApiController → JSON 응답
⬇️
5
JSON 데이터 수신
Sample API에서 날씨 데이터를 JSON 형태로 받음
{"location":"서울","temperature":"15°C","condition":"Sunny","humidity":"60%","windSpeed":"10 km/h"}
⬇️
6
순수 C# 코드로 응답 생성
AI 모델 없이 미리 정의된 템플릿으로 응답 생성
GenerateWeatherResponse() 메서드 실행 JSON 파싱 → 템플릿 적용 → 구조화된 응답
⬇️
7
최종 응답 반환
사용자에게 구조화된 날씨 정보 표시
🎬 실제 동작 예시
입력: "서울 날씨 알려줘"
서울의 현재 날씨를 알려드리겠습니다. 🌡️ 기온: 15°C ☀️ 날씨: Sunny 💧 습도: 60% 💨 바람: 10 km/h 좋은 하루 되세요!

🆚 Agent vs Ollama 모드 비교

🤖 Agent 모드
  • ✅ AI 모델 사용 안함
  • ✅ 순수 C# 코드
  • ✅ 빠른 응답 (< 1초)
  • ✅ 구조화된 템플릿
  • ✅ 100% 예측 가능
  • ❌ 자연스러운 대화 제한
  • ❌ 내부 HTTP 호출 있음
🧠 Ollama 모드
  • ✅ 로컬 LLM 사용
  • ✅ 자연스러운 대화
  • ✅ 창의적 응답
  • ❌ 느린 응답 (3-6초)
  • ❌ AI 모델 의존
  • ❌ 예측 불가능
  • ❌ Ollama 서버 필요
🔍 네트워크 의존성:
• Agent 모드: 내부 HTTP 호출만 (localhost:5152)
• Ollama 모드: 내부 HTTP + Ollama 서버 (localhost:11434)
• 둘 다 외부 인터넷은 불필요!
